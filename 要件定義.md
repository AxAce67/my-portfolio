# ポートフォリオサイト 要件定義書

## 1. プロジェクト概要

### 1.1 目的
個人開発者としてのポートフォリオサイト構築。プロジェクトの進捗管理、過去の実績紹介、スキルセットの可視化を行う。

### 1.2 ターゲットユーザー
- 採用担当者
- クライアント候補
- 技術コミュニティのメンバー
- 一般の訪問者

---

## 2. システム構成

### 2.1 技術スタック

#### フロントエンド
- **フレームワーク**: Next.js 14（App Router）
- **言語**: TypeScript
- **スタイリング**: Tailwind CSS
- **アニメーション**: Framer Motion
- **フォーム**: React Hook Form + Zod
- **多言語化**: next-intl（日本語/英語）

#### バックエンド
- **BaaS**: Supabase
  - データベース（PostgreSQL）
  - 認証（Supabase Auth）
  - ストレージ（画像管理）
  - Row Level Security（RLS）

#### インフラ・ツール
- **ホスティング**: Vercel
- **メール送信**: Resend（お問い合わせ通知）
- **スパム対策**: Cloudflare Turnstile
- **エラー監視**: Sentry
- **アナリティクス**: Google Analytics 4
- **CI/CD**: GitHub Actions

## 3. デザインコンセプト

### 3.1 テーマ
**モダン × シェル/ターミナルのハイブリッド**

- ベース: ミニマル、テック系
- アクセント: シェル/コード要素を効果的に配置
- カラースキーム: 白黒ベース + アクセントカラー（ターミナルの緑 `#00FF00` や青 `#00D9FF`）

### 3.2 背景デザイン
- **基本**: 白（ライトモード）/ 黒（ダークモード）
- **グリッドパターン**: 
  - 薄いグレーのドット or 細い縦横線
  - 静的（動かない）
  - 透明度: 5-10%（ほぼ見えない程度）
  - 背景が単調にならない程度の控えめな装飾
- **セクション区切り**:
  - 微妙なグラデーション（グレー → ダークグレー）
  - または薄い罫線（1px）

### 3.3 フォント
- **コード要素・見出し**: JetBrains Mono / Fira Code（モノスペース）
- **本文**: Inter / Geist（サンセリフ）
- **フォントウェイト**:
  - 名前・大見出し: 700 (Bold)
  - 肩書き・中見出し: 500-600 (Medium/Semi-Bold)
  - 本文: 400 (Regular)

### 3.4 カラーモード
- ダークモード・ライトモード切り替え対応
### 3.4 カラーモード
- ダークモード・ライトモード切り替え対応
- **ライトモード**: 
  - 背景: `#FFFFFF`
  - 文字: `#000000`
  - グレー（罫線・区切り）: `#CCCCCC`
  - アクセント: `#00FF00` または `#00D9FF`（ホバー時のみ）
- **ダークモード**: 
  - 背景: `#000000`
  - 文字: `#FFFFFF`
  - グレー: `#333333`
  - アクセント: `#00FF00` または `#00D9FF`

### 3.5 レスポンシブ対応
- モバイル、タブレット、デスクトップ全対応
- モバイルファースト設計

  - Skills

  - Contact

- ダークモード切り替えボタン

#### 4.1.2 Hero セクション
**デザインコンセプト**: ターミナル要素を控えめに、メインコンテンツは大きく見せる

**レイアウト**:
```
[画面上部、小さく装飾として]
> initializing...

[画面中央、大きく]
━━━━━━━━━━━━━━━━━
  あなたの名前
━━━━━━━━━━━━━━━━━

個人開発者 | Full Stack Developer

[タイピングアニメーションで表示]
好きなものを作る。それだけ。█

[下部]
$ cd projects/ ↓
```

**フォントサイズ**:
- 名前: `64px〜96px`（デスクトップ）、`40px〜56px`（モバイル）
- 肩書き: `24px〜32px`（デスクトップ）、`18px〜24px`（モバイル）
- 説明文: `18px〜24px`（デスクトップ）、`16px〜20px`（モバイル）
- ターミナル風装飾: `14px〜16px`

**アニメーション**:
- タイピングアニメーション（説明文が1文字ずつ表示）
- カーソル点滅（0.7秒間隔）
- フェードインアニメーション（要素が順番に表示）
- スクロールダウンアイコンの上下移動

**配色**:
- 背景: ページ全体と同じ（白 or 黒）
- 名前: 太字（font-weight: 700）
- アクセント: カーソル点滅、スクロールアイコンのみアクセントカラー
- ターミナル風の装飾（`>`, `$`）: グレー（控えめに）

**余白**:
- 上下に十分なスペース（画面高さの80-100vh）
- 左右の余白もたっぷり
- 息苦しくない、広々とした印象

## 6. 非機能要件

### 6.1 パフォーマンス
- ページ読み込み時間: 3秒以内（LCP）
- 画像の遅延読み込み（Lazy Loading）
- Next.js Image最適化活用
- ISR（Incremental Static Regeneration）でキャッシュ活用

### 6.2 SEO対策
- メタタグ最適化（title, description, OGP）
- 動的OGP画像生成（Next.js ImageResponse API）
- サイトマップ自動生成
- robots.txt設定
- 構造化データ（JSON-LD）
- セマンティックHTML

### 6.3 セキュリティ
- Supabase RLS（Row Level Security）によるアクセス制御
- CSRF対策
- XSS対策（入力値のサニタイゼーション）
- 環境変数の適切な管理（Vercel）
- HTTPS通信の強制

### 6.4 アクセシビリティ
- WCAG 2.1 AA レベル準拠
- キーボード操作対応
- スクリーンリーダー対応
- 適切なARIAラベル
- カラーコントラスト比の確保

### 6.5 ブラウザ対応
- Chrome（最新版）
- Firefox（最新版）
- Safari（最新版）
- Edge（最新版）
- モバイルブラウザ（iOS Safari, Chrome Mobile）

### 6.6 エラーハンドリング
- エラーバウンダリの実装
- ユーザーフレンドリーなエラーメッセージ
- Sentryによるエラー監視・通知
- フォールバック UI

### 6.7 バックアップ・復旧
- Supabaseの自動バックアップ活用
- 定期的なデータエクスポート

## 7. UI/UX 詳細仕様

### 7.1 アニメーション
- **ページロード時**:
  - ターミナル風ローディング表示（`Loading...`）
  - 0.3秒のフェードイン
- **Heroセクション**:
  - タイピングアニメーション（1文字ずつ表示、50-80ms間隔）
  - カーソル点滅（0.7秒間隔）
  - スクロールダウンアイコンの上下移動（無限ループ）
- **セクション**:
  - スクロールトリガーでフェードインアニメーション
  - 要素が順番に表示（stagger効果）
  - イージング: ease-out
- **カード**:
  - ホバーで浮き上がる（translateY: -8px）
  - 微妙な影の拡大
  - トランジション: 0.2s
- **ボタン**:
  - ホバーでアクセントカラーが光る（グロー効果）
  - クリック時の縮小アニメーション（scale: 0.95）
- **Tech Stackスクロール**:
  - 無限横スクロール（自動）
  - ホバーで一時停止
  - スムーズな繰り返し

### 7.2 インタラクション
- **ホバーエフェクト**:
  - リンク: 下線がスライドイン
  - カード: 浮き上がり + 影の拡大
  - ロゴ（Tech Stack）: グレースケール → カラー
  - ボタン: 背景色の変化 + グロー効果
- **クリックフィードバック**:
  - ボタン: 縮小アニメーション
  - カード: ripple効果（オプション）
- **スクロール**:
  - スムーズスクロール有効
  - セクションアンカーリンクで自動スクロール
  - プログレスバー（ページトップに細い線、オプション）

### 7.3 ローディング状態
- **初期ロード**: ターミナル風ローディング画面
  ```
  > Initializing...
  > Loading portfolio...
  > Done.
  ```
- **スケルトンスクリーン**: プロジェクト一覧読み込み時
- **プログレスインジケーター**: 画像アップロード時（管理画面）
- **ボタンのローディング状態**: スピナー + 無効化

### 7.4 トランジション
- **ページ遷移**: シンプルなフェード（0.2s）
- **モーダル開閉**: スケールアニメーション（0.3s）
- **タブ切り替え**: スライドアニメーション（0.2s）
- **フィルタ適用**: フェードアウト/イン（0.15s）

### 7.5 レイアウト
- 1ページスクロール型（セクションごとにアンカーリンク）
- 固定ヘッダー（ナビゲーション）
- フッター（SNSリンク、コピーライト）
